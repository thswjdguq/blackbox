# Task 06 — SMOKE_TESTS 베이스라인 1회 실행

## 1. 한 줄 요약
Task 05에서 작성한 SMOKE_TESTS.md의 7개 시나리오를 실제로 1회 실행하고 결과를 "실행 이력" 표 첫 행에 기록해 Phase 0 이후 회귀 비교 기준점을 만든다.

## 2. 변경 범위 (HARD LIMIT)

- **수정 파일:**
  - `docs/refactor/SMOKE_TESTS.md` — "실행 이력" 표에 1행 추가만
- **수정 라인 수:** 1줄 추가 (표 한 행) + 시나리오별 결과 코멘트(선택, ≤5줄)
- **예상 변경 파일 수:** 1개

**이 Task는 코드·다른 문서·시나리오 본문을 절대 수정하지 않는다.**

## 3. 절대 건드리지 말 것 (Out of Scope)

- SMOKE_TESTS.md의 시나리오 본문(S1~S7), 사전 준비, 정책 섹션 — 결과 표 1행 외 어떤 부분도 수정 금지
- 시나리오 실행 중 발견된 버그·UX 이슈는 **수정 시도 금지** → 실행 이력 표 비고에 기록 + Phase 4 후보로 별도 발의
- 코드·설정·다른 문서 일체 수정 금지

## 4. 컨텍스트 / 의도

Task 05는 시나리오 명세 작성. 본 Task는 **실제 통과 여부 검증**. 베이스라인 실행 결과가 없으면 후속 Phase에서 회귀 판정 불가.

- 관련 INV: 해당 없음
- 관련 Drift ID: 실행 중 발견되는 항목은 추후 D-RUN-XX로 등록
- 관련 PRINCIPLES 섹션: §0 (대전제), §11 (검수), §13 (오케스트레이터 패턴)

## 4-1. 의존 관계

- **선행 Task:** 02 (빌드 통과해야 docker compose up 가능), 05 (시나리오 명세 머지)
- **후행 Task:** Phase 2 작업 시 회귀 비교에 사용
- **API surface 변경 여부:** No

## 5. 적용해야 할 PRINCIPLES (본문 발췌)

> PRINCIPLES.md §13 (저수준 모델 대응 — 오케스트레이터 패턴) 발췌:

명령 실행 + 출력 캡처 Task의 패턴 ("오케스트레이터 + 사용자 실행"):
환각 위험이 본질적이라 Copilot에게 직접 시키지 않는다.
- Copilot은 "다음 명령 실행 후 결과를 붙여주세요" 안내만
- 실제 실행과 출력 복사는 사용자
- Copilot은 받은 출력을 문서로 정리

## 6. 작업 절차 (오케스트레이터 + 사용자 실행 패턴)

> **Copilot은 시나리오를 직접 실행하지 않는다.** 사용자에게 SMOKE_TESTS.md를 보면서 차례로 실행하도록 안내하고, 받은 결과를 표에 기록.
> 환경: **WSL Ubuntu bash + 브라우저** 기준.

### 6-1. 사전 안내

Copilot 발화 예시:
> "지금부터 SMOKE_TESTS.md의 7개 시나리오를 차례로 실행하시면 됩니다.
> 먼저 사전 준비 섹션의 명령으로 스택을 깨끗이 기동해주세요:
> ```
> docker compose down -v && docker compose up -d --build
> docker compose ps
> ```
> 모든 서비스가 healthy/running이면 알려주세요. 안 되면 그 출력을 붙여주세요."

### 6-2. 시나리오 S1~S7 순차 안내

각 시나리오마다 동일 패턴:
> "이제 시나리오 S{N} ({이름}) 실행 부탁드려요. SMOKE_TESTS.md의 단계 1~M 따라 진행하시고, 결과를 다음 4개 중 하나로 알려주세요:
> - ✅ pass (예상 결과대로 동작)
> - ❌ fail + 어느 단계에서, 무엇을 기대했고, 무엇을 보았는지 1~2줄
> - ⏭ skipped + 사유 (예: 환경 조건 미충족)
> - ⚠️ partial + 어느 부분이 부분 통과인지"

### 6-3. 실행 이력 표에 기록

7개 시나리오 결과를 모아 SMOKE_TESTS.md의 "실행 이력" 표에 1행 추가:

```markdown
| <YYYY-MM-DD> | <사용자 이름 또는 git user> | Phase 0 baseline | <S1결과> | <S2> | <S3> | <S4> | <S5> | <S6> | <S7> | <비고: 본 PR 링크 또는 발견사항 요약> |
```

비고 칸:
- 모두 ✅면 "전체 통과"
- 일부 ❌/⚠️면 "S{N} 이슈: <한 줄 요약>" + 별도 issue/Phase 4 Task 후보 명시

### 6-4. 발견된 이슈는 별도 발의

❌/⚠️로 나온 시나리오는 Phase 4 Task 후보(`70-fix-<short>`)로 별도 발의.
**이 Task에서는 수정 절대 금지.** 실행 이력 표 + 비고에 기록만.

### 6-5. 사용자 검증

Copilot 발화 예시:
> "실행 이력 표에 다음 행을 추가했습니다:
> [표 행 인용]
> 결과가 본인이 본 것과 일치하는지 확인 부탁드립니다."
